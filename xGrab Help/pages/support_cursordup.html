Cursor is duplicated if screen is zoomed


xGrab uses a blazing fast capture method which is implemented in Mac OS X by OpenGL and QuickTime. This method, however, does not provide images with cursors. xGrab circumvents this shortcoming by composing a cursor image on top of the capture buffer.

But if a user chooses to zoom in on the screen using ctrl-mouse wheel, then the capture buffer does include a cursor. xGrab is not aware of this not can it be because there are no notifications of current zoom level changes in Mac OS X. Because of this, xGrab always draws a cursor image regardless of whether the capture buffer already includes one.

Apple is aware of this problem and as soon as it is addressed, xGrab will be updated to correct the issue.

To solve this problem do one of the following:

  • Refrain from zooming in while capturing
  • If you feel the need to increase GUI relative appearance, consider reducing your screen resolution instead
